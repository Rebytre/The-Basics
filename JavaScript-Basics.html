Variables:

Variable Declarations:

In JavaScript, you can create or declare variables by using one of three keywords:

  • "var"
  • "let"
  • "const"
Each keyword has its own characteristics and use cases.
  • "var" is function scoped, meaning it is only accessible within the function it is declared.
  • "let" is block scoped, meaning it is only accessible within the block (a pair of curly braces) it is declared in.
  • "const" is also block scoped, but it cannot be reassigned its value once it is set, with the exception of the constant being an object or an array.

Examples:


// Declaring a variable using "var":
var a = 10;
console.log(a); // Outputs: 10

// Declaring a variable using "let":
let b = 5;
console.log(b); // Outputs: 5

// Declaring a variable using "const":
const apple = "red";
const apple = "green";
console.log(c); // Outputs: "red" (or an error, since there has been an attempt to reassign the variable when it is a constant)

Variable Scopes:

Similar to almost any other programming language out there, information is stored within a certain scope, meaning they can only be accessed within a certain area of the code.
There are three main scopes in JavaScript:

  • Global
  • Function
  • Block
Each of them are different in terms of where they can be accessed.
  • Global - Can be accessed anywhere within the code. A variable with a global scope can be created by declaring a variable outside of any function or block.
  • Function - Can only be accessed within the function it is declared in.
  • Block - Can only be accessed within the block it is declared in (a block being a pair of curly braces: {}).
Here is what scopes the different variable types can be accessed in:
  • "var": function level
  • "let": block level
  • "const": block level
And some questions that may arise:
If you were to declare any variable above its base scope, can you access it in any scope lower than the current scope?
Yes, any scope below that will be able to access that variable, whether it be "let", "var", or "const". For example: 

function example() {
    let x = 10 // The variable has a function scope

    if (true) {
        console.log(x + 5) // Despite "x" being in a higher scope, it can still be accessed within a lower scope.
    }
}

Variable Hoisting:

Variable hoisting refers to when a variable declaration will be compiled when running the code.
Only variables defined with "var" will be hoisted to the top of their scope.
Any other variables defined with "let" or const will not be hoisted.
One thing to note however, is that only the declaration of the variable will be hoisted, and not the initialization.
This means that if you try to access a "var" variable before it has been initialized, it will return "undefined".

Example:


console.log(x); // Outputs undefined, since it has been hoisted but not initialized.
var x = 45;
console.log(x); // Outputs 45, since the variable has now been initialized.

// Replace "var" with "let" or "const" and it will throw an error.

Data Types:

Strings:

Javascript strings are similar to every other string in other programming languages.
They are used to represent text and can be enclosed in single quotes (' '), double quotes (" "), or backticks (` `).
Single quotes and double quotes are functionally the same, but backticks allow for string "interpolation", and multi-line strings.
String interpolation allows you to embed expressions or other variables directly withing the string by wrapping the expression or variable in ${}.

Example:


let stringa = 'Hello!';
let stringb = "How are you?";
let stringc = `Daniel says: ${stringa} ${stringb}`; //String interpolation

console.log(stringa);
console.log(stringb);
console.log(stringc);

Booleans:

Booleans are a data type that can only have one of two values: "true" or "false".
They are most commonly used in conditional statements and loops to determine whether a block of code should execute.

Example:


let exectuteCode = true; // Replace with false to see other output.

if (executeCode) {
    console.log("The code has executed.");
} else {
    console.log("the code has not executed.");
}

Numbers:

Numbers in Javascript are used to represent both integers and other similar numberic values.
They can be used in any mathematical operations. Here is a list of the different types of number variables:

  • Integers - Whole Numbers eg. 1, 2, 3, ...
  • Floats - Decimal Numbers eg. 1.58, 0.2, 3.14159, ...
  • Hexadecimal Notation - Base 16 Numbers eg. 0xff, 0x5a, 0x2b2, ... (Make sure to always start the value with "0x")
  • Binary Notation - Base 2 Numbers eg. 0b1, 0b101011, 0b11111110, ... (Make sure to always start the value with "0b")
  • Exponential Notation - Numbers written in the form of a * 10^b eg. 1.5e2, 7.2e50, 12.2e-10, ... (Make sure to always end the value with "e" followed by the exponent value)

Quick note: Javascript regards all of these values as being a "number" value, meaning if you use a "===" equality comparison operator, it will return true if the values are equal, regardless of the format of the number variable.

Example:


let intnum = 42; // Integer Type
let binarynum = 0b101010; // Binary Notation Type

if (intnum === binarynum) {
    console.log("The two numbers are equal.");
} else {
    console.log("The two numbers are not equal.");
}
